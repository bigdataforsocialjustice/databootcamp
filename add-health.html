Add Health

Add Health data is one of the most widely utilized datasets in public health. Add Health is the largest, most comprehensive, nationally-representative longitudinal survey of adolescents ever undertaken. Beginning with an in-school questionnaire administered to a nationally representative sample of students in grades 7-12, the study followed up with a series of in-home interviews conducted in 1995, 1996, 2001-02, 2008, and 2016-18. As a result, there are 5 waves of publicly available data that are can be downloaded and analyzed.

We are going to learn how to clean, merge and analyze data using multiple waves of the Add Health survey.

We don't have time to go into details about how to formulate a research question. However, you should be able to come up with many ideas by looking across waves.

With that said, I am interested in the following questions: (1) Are women who have child maltreatment histories more likely to have poor birth outcomes as adults?; and (2) Does the impact of maltreatment history differ across race/ethnicity. Answering this question using these data is not straightforward. However, we can do it by merging the datasets. First, however, some pretty significant data cleaning needs to be done.

Note that you will not know which of the 42 datasets may contain information that we need to address these questions. Let's open the datasets in SPSS and identify the fields we want to analyze. Because of the magnitude of this dataset, I will create a subset to work with.

To locate the variables pertaining to child maltreatment we identified above, from the 'Variable View' tab click control-F to invoke the search dialog. Then, type in the variable name from above to look for the 'times left alone variable': H3MA1. The search dialog will take you directly to that variable. This will save you a ton of time.

Activity: search for the race variable(s). We will also include the Respondent identifier, Biological Sex and Race in our subset.

To subset, from the file menu go to File --> Save As --> and then search for the variables that you want to save (i.e., the maltreatment variables, race, R identifier and bio sex). You will notice that it takes a long time to identify the variables.

Here is another trick. From any menu in SPSS you can sort the fields by alphabet or file order. Sometimes we know the file order and it's easier to sort on that. However, oftentimes its more intuitive to sort the variable names alphabetically. To quickly identify the variable we want to subset, let's rename them to facilitate their identification.

Now, go to File --> Save As --> Variables --> Drop All and click 'Name' once in the dialog box. All of our variables rise to the top of the dataset.


Save the file as 'W3_inhome_subset.sav'

Let's now take a look at the Pregnancy Outcomes dataset. Unfortunately, this data is not as straightforward. First, notice that the field AID also appears in this dataset. This field is the unique respondent identifier that you need to use in order to merge the data across datasets. The reason that this is not straightforward is because the AID field contains duplicates. Common sense would suggest that each row of the data corresponds to a different pregnancy, and that is the case. If we look at the first two columns, we see that the first respondent AID has four rows and four pregnancies.

Let's subset this data as well, so we can focus on 2-3 pregnancy outcomes.
The next step is to merge the data so that we have only one file containing the maltreatment variables, race, biological sex, and all of the pregnancy outcomes we want to analyze, all linked by the unique respondent ID. Based on my limited knowledge, I am going to choose preterm birth H5PG15, baby birth weight H5PG13L. Subset these variables (don't forget to include AID) and call the new file W5_birth_outcomes.

We should stop here. We are focusing on the steps needed to get data in a form we can use. In real life, it is important to think about theory and other variables that need to be included in the analysis (i.e., confounding variables, moderators and mediators). For example, theoretically, why would we expect child maltreatment to impact birth outcomes? Would this explanation apply to only first births, subsequent births, etc.? What other information would be important to include in this analysis? At minimum, wouldn't we want to know about the health status of the mother? prenatal care? smoking behaviors? presence of social support? Luckily, all of these variables are included in Add Health and can be added to our models.

    Now that we have our dataset, let's see what variables need to be cleaned. Run Analyze --> Descriptive Statistics --> Frequencies and move all variables into the Variable(s) dialog box. It appears that the variable preterm birth has not been coded for missingness. Value 98 means that no information was given for that respondent. To tell SPSS that 98 is a missing value, go the the 'Variable View' and locate the variable 'preterm.'

    Click inside the missing column for that variable and select the three dots. Type in '98' as a discrete missing value and re-run the descriptive statistics to check that it is correct.

Now our data is almost ready to be analyzed. But first, note that the unit of analysis is no longer the individual, what is it? What we may want to do is aggregate average birthweight across all pregnancies and count the number of preterm births for each respondent before doing the analysis. This puts the unit of analysis back to the individual level.

Unfortunately, without resorting to code, we cannot aggregate the data in the way that I want. Fortunately, we can do this really easily in excel. Save this file as an xlsx file and open it in Excel.
